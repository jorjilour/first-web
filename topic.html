JavaScript Object Notation (JSON)

JSON Definition and its Purpose

Javascript Object Notation, also known as, JSON, is a lightweight data-interchange format i.e., it is a means to store and exchange information in an organized and easy-to-access manner. JSON is designed in a way where it can display a collection of data as text, although it adapts the Javascript syntax, it can easily be understood or read by humans and easily be parsed and generated by machines as well. As it is stored and/or sent as a text, it can effortlessly be sent to and from a server and it can be read and used as a data format by ANY programming language, making it language-independent

The JSON Saga

JSON's Origin

The JSON format was discovered, specified, and popularised by Douglas Crockford, along with his State Software team, a company he co-founded where he was the CTO at that time.

However, Crockford declares that he was not the one who invented this type of format,

“I do not claim to have invented JSON. It already existed in nature. I do not claim to have been the first to discover it. I found it, named it, and described what it was used for. I gave it a specification and a little website.”

The very first JSON message was made in April 2001 embedded in an HTML document.

JSON is a standard format: the ECMA-404 The JSON Data Interchange Standard and specified in the domain name he registered, www.jason.org, in 2002. In 2005, the need for an alternative data interchange format to operate smoothly and continuously, during the emerging Single Page Applications and modern mobile/web apps, expanded its popularity and boost its usage.

Why JSON?

JSON Features

  • Easy to use - JSON API offers high-level facade, which helps you to simplify commonly used use-cases.
  • Performance - JSON is quite fast as it consumes very less memory space, which is especially suitable for large object graphs or systems.
  • Free tool - JSON library is open source and free to use.
  • Clean JSON - Creates clean, and compatible JSON result that is easy to read.
  • Dependency - JSON library does not require any other library for processing.

JSON's Growth and Future

Today, JSON has been adopted by most popular web APIs, like Google, Facebook, and Twitter. JSON has also been widely adopted and questions being asked, have increased in Stack Overflow, a question and answer site for programmers.

JSON’s simplicity allowed it to gain rapid popularity as it saves storage space and makes data transfer fast and easy.
In the foreseeing future, it is believed to remain a big player as a data interchanged format. For the time being, JSON appears to be the go-to format for communicating with other programs over the internet.

Basic Syntax

JSON is built on two structures. A collection of data in JSON format may contain:

  1. An object containing a collection of name/value pairs.

    This object refers to the unordered set of the name/value pairs. This object starts with a left brace “{“ and ends with a right brace “}”. Each name is followed by a colon, “:”and the name/value pairs are separated by a comma, “,”.

    { “name” : “Jorjilou”,
    “age” : 22,
    “nickname” : “Juday”

  2. An ordered list of values stored as an array or a list.

    An array refers to a collection of values that is ordered. It should begin with a left bracket , “[“ , and end with a right bracket, “]”.

    For example: the value section from the name/value pair in the object below is an array, which is a collection of numbers.
    {"ages":[22,23,24,25,26]}

The structures mentioned above can be nested.

A value can either be a string in double quotes, a number (integer, float, digits etc), another object [1], an array, boolean values (true or false), or null.
